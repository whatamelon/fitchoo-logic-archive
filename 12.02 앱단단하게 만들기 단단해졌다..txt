12.02	앱 단단하게 만들기

	1. 나갈때는 로딩이 돌면 안되고 들어올 때는 돌아도 된다.
	2. 트렌지션이 우선이지만, 부득이 하게 로딩이 돌아야 한다면, 로딩이 돌지 않도록 앱을 잘 만들자.
	3. 트렌지션과 로딩이 함께 도는 UX는 없다.


	이제 점검해보자.

	0. 전체적으로 찜 삭제 할 때 왜 로딩이 도는지 모르겠다. -> 이게 꼭 필요한 건지 모르겠음.

	1. 팔로우들어갈때 로딩 돈다잉 / 팔로우 -> 모델상세 트렌지션 ㅇㅋ / 모델상세 -> 팔로우 트렌지션 x 로딩 돈다.
		-> 로딩빼고 트렌지션으로 고치기. ( O )

	2. 찜 + 최근본 상품 들어올때 & 나갈 때 로딩돌고, 심지어 트렌지션도 같이 돈다. / 마이페이지 갈때는 로딩 x
		-> 얘네도 로딩 다 빼고 트렌지션으로 고치기
		-> 페이지 <-> 상품상세 이동시 로고 나오는데 이거 고치자  ( O )

	3. 상품상세 -> 상품이 바뀌는 로직 발생시, 미리 바뀌고 페이지 이동이 된다.
		-> 이거 이동하고 바뀌는 걸로 아님 이동중 바뀌는 걸로 고치자
		-> 이건 트렌지션의 우선순위와 관련이 있음
		-> 트렌지션은 기본적으로 받는 페이지에서 실행됨.
		-> 반면 상품을 클릭하면, 비동기적으로 먼저 상품을 가져오는 로직이 실행되기 때문에,
			ux 로 보면, 이상한게 맞음
			방법이 있을 것. ( O ) v-once로 해결

	4. 컬렉션 나갈 때 또 상품들 다 없어진 상태가 되는데 이거 해결하자. + 찜도 이 문제 있음.
		-> 찜은 이거 때문에 로딩돌게 한 듯. ( O )

	5. 모델 상세 -> 상품 상세 이동시 트렌지션 안먹는다.
	
	6. 상품 상세 -> 모델 상세 이동시 로고 보인다. 해결하자. ( O )
